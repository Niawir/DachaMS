# DachaMS

Система управления дачным/коттеджным сообществом — учебный проект.

## Стек
- **C# WinForms** — интерфейс
- **MS SQL Server** — база данных
- Visual Studio Designer

## Возможности

### Роли и доступ
- **Admin** — полный доступ ко всем разделам, управление пользователями
- **Moderator** — расширенные права на редактирование
- **Viewer** — доступ только на чтение (по умолчанию для новых пользователей)
- Защита от смены собственной роли, хеширование паролей (SHA-256)

### Интерфейс
- `MainForm` — боковое меню, рабочая область, кнопка "Назад", ФИО и роль пользователя в шапке
- `LoginForm` — вкладки "Вход" / "Регистрация"
- Все панели открываются внутри `MainForm`, без отдельных окон

### Разделы
- **Участки** — с отображением цены прямо в таблице, история изменений по участку
- **Владельцы** — привязка владельцев к участкам
- **Постройки** — учёт построек на участках
- **Правила цен** — настройка ценообразования по зданию (базовая цена, бонусы, электричество, статус заброшенности)
- **Правила рейтинга** — настройка баллов рейтинга (состояние, электричество)
- **Рейтинг** — расчёт и отображение рейтинга участков
- **Статистика** — 16 метрик по сообществу
- **Пользователи** — управление учётными записями (только admin)

## База данных

Нормализованная схема на T-SQL:
- `users` (роли: admin / moderator / viewer)
- `owners`
- `buildings`
- `price_rules` (building_id, база, бонус, электричество, статус заброшенности)
- `rating_rules` (building_id, бонус рейтинга, баллы за электричество/состояние)
- `plots`
- `plot_buildings` (связующая таблица участков и построек)

## Дополнительно
- Генерация отчётов в формате Word по ГОСТ
- Отдельный вариант с нормализованной 5-сущностной схемой (Отделения, Врачи, Пациенты, Диагнозы, История болезни) с CRUD-панелями на DataGridView
